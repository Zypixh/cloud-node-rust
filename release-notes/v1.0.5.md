# cloud-node-rust 1.0.5

本文件记录 `1.0.5` 的发布说明。

## 修复与核心优化

- **修复底层长连接并发抽取竞态 Bug (Pingora Core)**
  - **核心修复**：修复了 Pingora 核心连接池中 `Arc::try_unwrap` 的竞态条件，该问题导致上游复用连接被错误地判定为获取失败，引发 `failed to acquire reusable stream` 报错。
  - **退让重试**：通过引入带最大阈值的 `tokio::task::yield_now().await` 异步退让机制，确保连接被彻底释放和引用计数归零后再提取底层流，完美接管并复用上游连接。

- **修复 HTTP/1.1 Keep-Alive 长连接意外断开问题**
  - **管道复用**：修复了 HTTP/1.1 代理循环在处理完首个请求后直接退出协程，导致客户端 Keep-Alive 连接被强行切断的 Bug（表现为高并发压力测试下大量的 `connection error`）。
  - **读写超时阻断**：增加了对上游请求强制的 60 秒 `read_timeout` 和 `write_timeout`。这防止了由于后端无响应或半开连接而导致 Tokio 工作线程 (Workers) 被无限期挂起，有效提高了极端网络条件下的整机吞吐量和并发处理性能。
  - **循环消费**：现已支持基于底层返回的 `ReusedHttpStream` 进行 `while` 循环持续消费流，完美支持客户端的连接复用，使压测成功率追平并对标 Go 原版实现。
  - **空闲超时**：主动补充了 60 秒的空闲连接回收机制，防止无脑挂起的僵尸连接耗尽系统句柄。

- **修复缓存锁 (Cache Lock) 导致的 `wrong phase: Uninit` Panic 崩溃**
  - **参数整合**：修复了 Pingora 内核缓存模块的状态机校验崩溃。原逻辑在调用 `session.cache.enable()` (使状态进入 `Uninit`) 后强行调用 `set_cache_lock()`，触发了严格的 Phase 检查 Panic。现将全局防击穿锁 `CACHE_LOCK` 直接作为 `enable()` 的内联参数传入，彻底消除了由于状态机时序错误导致的工作线程崩溃现象。
- **修复 UDP 代理的严重任务与内存泄漏**
  - **生命周期解耦**：在 `src/udp_proxy.rs` 的 UDP 转发任务中，去除了对外层 `Arc<UdpSession>` 的闭包强行捕获，改为值传递核心参数。
  - **精准回收**：彻底解决了由于 `mpsc::Sender` 被异步任务无限期持有，导致 `rx.recv()` 永远无法结束从而引发的协程卡死与内存泄漏。现在 UDP 空闲绘画超时后，将立刻精准阻断并销毁后端任务。

- **大幅降低 CPU 消耗与 TLS 握手瓶颈 (TLS Acceptor Optimization)**
  - **单例实例化**：移除了 `http_proxy_manager` 和 `tcp_proxy` 中在每一个新连接到来时都去实例化 `SslAcceptor::mozilla_intermediate_v5` 构造器并建立 OpenSSL 上下文的严重高耗时操作。现在 SSL Acceptor 会在 Listen 协程启动时构建一次并套入 `Arc` 跨连接复用。这一改进将大幅降低突发高并发请求下的 CPU 消耗，消除锁竞争，极大提升了新建 HTTPS/TLS 连接的吞吐率（QPS）。
  - **消除栈上巨型数组**：深挖并优化了存在于 HTTP TLS SNI 嗅探 (`64KB`)、TCP 数据拷贝 (`16KB`) 和 UDP 数据透传 (`64KB`) 核心流程中的局部巨大数组。
  - **强制堆分配**：统一将类似于 `[0u8; 65535]` 的局部变量改为 `vec![0u8; ...]` 堆分配。这一举措大幅度缩减了 Tokio 异步函数编译后生成的底层状态机 (Future) 的结构体大小，彻底排除了在几万并发下由于协程栈内存累加吃满数 GB 导致节点 OOM 崩溃的定时炸弹。
