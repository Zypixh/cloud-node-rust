# cloud-node-rust 1.0.4

本文件记录 `1.0.4` 的发布说明。

## 修复与优化

- **重构目录层级结构**
  - **对齐规范**：将所有持久化数据路径由 `data/` 调整为 `../data/`。
  - **目录分离**：建议将二进制文件存放在 `bin/` 目录下，使 `data/` 目录位于其上级，与原 Go 版本实现保持一致。

- **实现健壮的单实例保护机制**
  - **文件锁机制**：在 `src/main.rs` 中引入了基于 `libc::flock` 的建议性文件锁。
  - **自动释放**：利用操作系统特性，确保程序在 `Ctrl+C`、被 kill 或崩溃时能自动释放锁，彻底解决了旧版本 PID 文件残留导致无法重启的问题。
  - **跨平台支持**：支持 Linux 和 macOS，避免了对 `/proc` 目录的硬编码依赖。

- **修复 RUST_LOG 环境变量不生效的问题**
  - **环境过滤**：引入了 `EnvFilter`，使程序能够正确识别并遵循 `RUST_LOG` 环境变量的设置（如 `export RUST_LOG=off`）。
  - **静默启动**：现在可以通过环境变量完全关闭控制台启动信息及运行日志。

- **重构 gRPC 与 WebSocket 的兼容性逻辑**
  - **支持联动**：恢复了“开启 WebSocket 自动激活 gRPC 支持”的业务逻辑。
  - **智能协议识别**：仅在请求内容确认为 `application/grpc` 时才会强制使用 HTTP/2 (H2) 回源，解决了 H1.1 WebSocket 的 502 问题。

- **优化日志缓存标签上报**
  - **精准上报**：访问日志中移除了 `X_CACHE_BYPASS`、`X_CACHE_MISS` 等非命中状态的标签。
  - **命中触发**：仅在 HIT、STALE 或 REVALIDATED 时记录 `cacheStatus` 及 `X_CACHE_` 标签。
  - **保持兼容**：HTTP 响应头中的 `X-Cache` 字段保持全量输出，方便调试。

- **屏蔽高频噪音日志**
  - **静默断开**：通过 `suppress_error_log` 屏蔽了由于客户端中途断开（Broken pipe）导致的下游写错误日志。

- **移除硬编码的缓存击穿防护限制**
  - **规则驱动**：彻底移除了隐式的 10 QPS 限制，所有拦截逻辑现在完全由 WAF 规则显式定义。

## 自动构建说明

- **关于构建耗时（约 15 分钟）的说明**
  - **高性能优化**：由于启用了 `LTO = fat` 和 `codegen-units = 1`，链接阶段耗时较长，以换取最高的运行效率。
  - **缓存优化**：已同步 `Cargo.lock` 版本，后续增量构建速度将大幅提升。
