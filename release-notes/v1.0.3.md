# cloud-node-rust 1.0.3

本文件用于记录 `1.0.3` 的增量更新，当前尚未发布 release。

## 增量更新

- 修复 `allowLANIP` / `allowLanIP` / `allowLanIp` / `allow_lan_ip` 的兼容解析
  - 解决控制面已启用局域网源站，但运行时仍打印 `Allow LAN Origin IP: No` 的问题
  - 新增宽松布尔解析，兼容：
    - `true/false`
    - `1/0`
    - `yes/no`
    - `on/off`
    - `enabled`

- 调整 GitHub 自动构建 runner 基线
  - release workflow 调整为 `ubuntu-22.04`
  - 目的：适配 **Debian 12 (glibc 2.36)** 的运行环境
  - 兼容性：生成的二进制文件基于 glibc 2.35，可完美运行于 Debian 12、Ubuntu 22.04+ 及其他现代发行版，同时保持较新的工具链特性

- 修复共享 `443` 端口下 TLS 回调导致的进程崩溃
  - 问题出现在证书/OCSP 回调里对 `tokio::sync::RwLock::blocking_read()` 的使用
  - 在 Tokio runtime worker 线程中，这会触发 `Cannot block the current thread from within a runtime`
  - 又因为回调运行在不能 unwind 的 C 边界，最终导致进程直接 abort
  - 现在 `DynamicCertSelector` 内部已改为同步 `std::sync::RwLock`
  - 修复后，`SNI passthrough` 与普通 TLS 站点共用 `443` 时不再因该回调路径崩溃

- 修复 `@sni_passthrough` 在共享 `443` 端口上的命中错误
  - 之前 passthrough 只通过 `host -> 单个 ServerConfig` 的映射查找
  - 当同域名存在普通 L7 站点占用映射时，passthrough 站点会被遮蔽
  - 结果是原始 TLS 流量落进普通 HTTPS/L7 解析，触发 `InvalidHTTPHeader`
  - 现在改为专门扫描并匹配：
    - `@sni_passthrough`
    - 域名
    - HTTPS 监听端口
  - 同时支持精确域名、通配符域名和 `sub_names`

- 调整 `@sni_passthrough` / L4 统计时机，改为边转发边记账
  - 之前 `tcp_proxy` 与 `SNI passthrough` 主要在连接结束后一次性累加流量
  - 在 AnyTLS / 长连接场景下，这会导致：
    - 最近带宽被连接断开瞬间冲高
    - 即时流量统计长期接近 0
    - 回源带宽样本缺失
  - 现在改为在双向转发过程中按 chunk 增量记录：
    - 站点下行/上行流量
    - 回源上行/下行流量
  - 该调整已提交，但当前尚未完成线上统计结果验证

- 调整 `ServerBandwidthStat` / `ServerDailyStat` 字段语义对齐旧 EdgeNode
  - `ServerBandwidthStat.totalBytes`
    - 从“上下行累计”调整为“仅下行累计”
  - `ServerDailyStat.bytes`
    - 从“下行 + 回源流量混合”调整为“仅下行流量”
  - 目的是避免 API 端基于这些字段计算时出现峰值带宽偏大、当日流量异常等问题
  - 该调整已提交，但当前尚未完成控制面展示结果验证

- 调整 `ServerDailyStat` 上传时机，改为按 5 分钟窗口切换聚合上传
  - 之前更接近“进程启动后的固定周期上报”
  - 现在按 `timeFrom/timeTo` 归入当前 5 分钟窗口聚合后再上传
  - 目的是改善：
    - 当天流量长期为 0
    - 当天/当月峰值宽带不更新
    - 最近回源带宽无数据
  - 该调整已提交，但当前尚未完成线上统计结果验证

- 调整 `NodeValue` 上报项，强制保留基础实时流量指标
  - 即使控制面配置未显式选择，也会保留这些基础项：
    - `trafficIn`
    - `trafficOut`
    - `allTraffic`
    - `requests`
    - `attackRequests`
  - 目的是降低“即时统计页面无数据”的概率
  - 该调整已提交，但当前尚未完成控制面展示结果验证

- 修复 `LogUploader` 与 `NodeLogUploader` 在没有日志时导致 CPU 100% 空转的死循环问题
  - 原因是当日志队列为空时，未能正确更新 `last_flush` 的时间，导致 `tokio::time::sleep_until` 传入过期时间从而立即返回并陷入死循环
  - 现已确保在每次超时触发后无条件更新 `last_flush` 时间

- 优化热路径性能，消除巨型 `ServerConfig` 的全量深拷贝开销
  - 将 `NodeConfig` 中管理的 `ServerConfig` 从按值存储重构为按原子引用计数 `Arc<ServerConfig>` 存储
  - 移除了贯穿 HTTP、TCP、UDP 代理层的 `.cloned()` 获取方式，改为 O(1) 的引用克隆
  - 极大降低了极高并发下的堆内存分配释放（Malloc/Free）开销与内存颠簸问题

- 对齐 EdgeNode 时间与统计上报格式
  - 控制台日志改为本地时区时间，避免默认 UTC `Z` 时间造成排障误判
  - 访问日志时间改为请求开始时间，字段 `timestamp` / `msec` / `timeISO8601` / `timeLocal` 与 EdgeNode 口径一致
  - 移除使用配置同步响应 `timestamp` 修正全局时间的逻辑，避免将配置版本时间误当作当前时间
  - 统一节点日志、节点状态、节点值、WAF/IP 过期、缓存元数据、统计窗口等时间入口
  - `HTTPAccessLog.statusMessage` 调整为空字符串，对齐 EdgeNode

- 修复统计上报的数据完整性问题
  - 新增带宽统计失败重传：`ServerBandwidthStat` 上传失败后保留 20 分钟并随后续窗口重试
  - 新增日流量统计失败重传：`ServerDailyStat` 与 `domainStats` 上传失败后保留 20 分钟
  - 新增 HTTP 域名日统计：按 `serverId + 5分钟窗口 + host` 聚合，上传时每个 server 取请求数 Top 20
  - 新增独立 IP 统计 tracker：按 `serverId + day + ip` 统计，避免带宽与日统计 reporter 竞争清空同一个 IP 集合
  - `ServerDailyStat` 主上报字段收敛到 EdgeNode 使用的字段，不再额外填充 `day/hour/timeFrom/timeTo/countIPs`

- 修复维度统计与 Top IP 上报缺失
  - HTTP 请求结束时写入自定义指标聚合、HTTP 请求维度聚合与 Top IP 统计
  - 将自定义指标聚合器与 HTTP 请求维度聚合器拆分，避免两个 reporter 消费同一批样本导致丢数
  - `MetricStat` 的 week 时间键由 `%Y%U` 近似调整为 ISO week `YYYYWW`
  - 当控制面未下发 `metricItems` 时主动清理自定义指标聚合，避免无配置场景下内存积压

- 启用本地统计持久化 flusher
  - 启动 `metrics::start_persistence_flusher()`，使 RocksDB 增量统计落盘逻辑实际运行
